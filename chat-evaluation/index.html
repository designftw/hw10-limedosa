Chat Evaluation

Overview:

There are many aspects of heuristic design that are missing from the original chat app. These such failures include but are not limited to:

  • Match between system and the real world

    The chat app uses a lot of jargon that is not common in most texting apps. A lot of the initial wording such as "channels" make it difficult to navigate since users are only given a text box and don't know what to type. To adress this, wording for the most things was altered.

    To match the system and the real world, emojis were included next to the buttons.

  • User Control and Freedom

    Users are able to have a lot of freedom on how they use their app. The things they can now edit include:

    • Texts they sent
    • Username
    • Display name
    • Who they want to text either throgh ActorID or through username

  • Consistency and Standards

    There was a lot of CSS added in order to make the buttons and feedback consistent throughout the app. These changes included:

    • making the "edit username" and "edit name" buttons the same
    • making the public forum or direct message the same.
    • making the "send" button stand out
    • ensuring all the buttons give constant feedback, especially when being hovered over.

  • Error Prevention

    There are many ways in which the user can make mistakes. Although these errors already had some sort of feedback, there were changes implemeted to optimize Error Prevention. These include:

    • getting alerts when:
      • username is changed
      • name is changed

  • Aesthetic and minimalist design

    There was a lot of CSS added to all features to make this appear more like a texting app.
